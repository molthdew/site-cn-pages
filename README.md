# site-cn-pages

本仓库用于归档和发布多个独立的 HTML 页面。项目不针对任何特定域名、网站或服务，仅作为静态页面的集合与分发点。

## 项目简介

`site-cn-pages` 是一个以存档为目的的静态页面仓库。其中存放的 HTML 文件均为独立页面，彼此之间无依赖关系。仓库的主要功能是提供文件的版本管理、历史追溯以及通过 GitHub Pages 进行简单的静态托管。

## 目录结构

```
/
├── index.html          # 仓库首页（可选）
├── pages/              # 主要页面存放目录
│   ├── page-1.html
│   ├── page-2.html
│   └── ...
├── assets/             # 公共资源（CSS、JS、图片等）
│   ├── css/
│   ├── js/
│   └── images/
└── README.md           # 本文件
```

- `pages/`：存放独立 HTML 页面的主目录。每个页面为一个独立的 HTML 文件。
- `assets/`：存放页面所需的静态资源文件，按类型分类存放。
- 仓库根目录下的 `index.html` 为可选的导航或说明页面。

## 页面归档说明

- 每个 HTML 页面均为自包含文件，尽量内联样式与脚本，以减少外部依赖。
- 页面归档不涉及任何动态后端服务，所有内容均为静态。
- 页面内容可能来源于历史存档、备份或技术演示，不保证实时性、准确性或适用性。
- 归档页面仅供参考、学习或个人使用，请勿用于商业或违规用途。

## 维护说明

- 维护者不定期向仓库中添加新的页面或更新已有页面。
- 提交历史中会注明每次变更的简要说明。
- 欢迎通过 Issue 提交建议或报告问题，但不对回复时间做出承诺。
- 不接受包含恶意代码、侵权内容或违反平台规则的文件提交。

## 使用方式

1. 克隆仓库到本地：
   ```bash
   git clone https://github.com/your-username/site-cn-pages.git
   ```
2. 直接在浏览器中打开 `pages/` 目录下的 HTML 文件即可查看。
3. 若启用 GitHub Pages，可通过仓库设置中的 Pages 选项选择分支和目录进行发布。
